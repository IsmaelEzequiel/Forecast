## Codebase Patterns
- Mix project root is at `weather_edge/` subdirectory (not repo root)
- Run `mix compile --warnings-as-errors` from `weather_edge/` for typecheck
- Oban cron workers are defined in `config/config.exs` - worker modules must exist before compilation
- Test config uses `Oban, testing: :inline` to disable async job processing
- Polymarket credentials are in runtime.exs under `:polymarket` key
- Trading safety config is under `:trading` key, forecasts under `:forecasts` key
- Docker: `docker-compose up` from `weather_edge/` dir; app runs migrations on startup via entrypoint.sh
- Production release uses `WeatherEdge.Release.migrate/0` for Ecto migrations
- Schemas follow `lib/weather_edge/<context>/<schema>.ex` pattern (e.g., `stations/station.ex`)
- Migrations use `timestamps(type: :timestamptz)`, schemas use `timestamps(type: :utc_datetime)`
- Hypertable migrations use `up/down` (not `change`), composite PK via ALTER TABLE, and `SELECT create_hypertable()`
- Trading context lives in `lib/weather_edge/trading/` (Position, Order schemas)
- Signals context lives in `lib/weather_edge/signals/` (Signal schema)
- Calibration context lives in `lib/weather_edge/calibration/` (Accuracy schema)
- Git working directory is `weather_edge/` - use relative paths for git add/commit
- Probability context lives in `lib/weather_edge/probability/` (Gaussian module)
