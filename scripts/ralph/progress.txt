## Codebase Patterns
- Phoenix LiveView app at weather_edge/ subdirectory
- Use WeatherEdgeWeb, :live_view for all LiveView modules
- Header component: import WeatherEdgeWeb.Components.HeaderComponent, use .dashboard_header
- PubSub: use WeatherEdge.PubSubHelper for subscribe/broadcast
- Signals table is TimescaleDB hypertable with: station_code, market_cluster_id, computed_at, outcome_label, model_probability, market_price, edge, recommended_side, alert_level
- Existing helpers in analytics_live.ex: format_pnl, format_price, format_pct, format_edge, side_class, alert_class, confidence_class, pnl_color, edge_color
- Dark mode support: always include dark: variants for zinc colors
- Tailwind styling: rounded-lg border border-zinc-200 dark:border-zinc-700 bg-white dark:bg-zinc-900 p-4
- Router auth: pipe_through [:browser, WeatherEdgeWeb.Plugs.RequireAuth]
- Balance from persistent_term: :persistent_term.get(:sidecar_balance, nil)
- Sidecar positions: :persistent_term.get(:sidecar_positions, [])
- Trading client: Application.get_env(:weather_edge, :clob_client, WeatherEdge.Trading.ClobClient)
- Existing contexts to reuse: WeatherEdge.Signals, WeatherEdge.Markets, WeatherEdge.Stations, WeatherEdge.Forecasts, WeatherEdge.Trading
- Job tracking: WeatherEdge.JobTracker.start/finish for worker state
- SignalsLive default_filters: %{stations: [], min_edge: 8, resolution_date: "all", side: "all", max_price: nil, alert_level: "all", sort_by: "edge_desc", actionable_only: false, has_position: "all"}
- Signals.Queries: list_filtered_signals(filters, opts) and count_filtered_signals(filters) for all filtered signal loading
- Filter changes in SignalsLive: use `reload_signals/2` (returns {:noreply, socket}) or `do_reload_signals/2` (returns socket) for handle_info contexts
- OrderManager.place_buy_order(station_code, outcome_map, amount) - outcome_map needs: "token_id", "outcome_label", "price", "market_cluster_id", "event_id"
- Station codes loaded from `Stations.list_stations()` on connected mount for filter dropdown
- Signal deduplication: DISTINCT ON (station_code, outcome_label, market_cluster_id) ordered by computed_at desc
- Position left_join: on market_cluster_id + outcome_label + status "open"
- Price trend: `Markets.price_trend(cluster_id, outcome_label, hours: 6)` returns `{:up|:down|:flat, delta_float}`

## 2026-03-07 - US-002
- Created WeatherEdge.Signals.Queries module with list_filtered_signals/2 and count_filtered_signals/1
- Deduplication via subquery: latest signal per (station_code, outcome_label, market_cluster_id) using DISTINCT ON + ORDER BY computed_at DESC
- Joins: Signal -> MarketCluster (market_cluster_id), Signal -> stations table (station_code), left_join positions (market_cluster_id + outcome_label + status "open")
- Base filter: mc.resolved == false
- Dynamic filters: stations, min_edge, side, max_price, alert_level, resolution_date (today/tomorrow/+2d/+3d), has_position, actionable_only
- Sort options: edge_desc, edge_asc, model_prob_desc, time_to_resolution, price_asc, newest
- Pagination via limit/offset (default 20/0)
- Returns maps with keys: signal, cluster, station, position, hours_to_resolution
- Files changed: lib/weather_edge/signals/queries.ex (new)
- **Learnings for future iterations:**
  - Position join needs station_code-less matching: join on market_cluster_id + outcome_label + status only (station_code already implied by cluster)
  - Station join uses raw table name "stations" with `on: st.code == s.station_code` since Signal doesn't have belongs_to :station
  - Ecto `[..., p]` syntax for referencing last binding in positional where clauses
  - `filter_actionable_only` compares s.market_price <= st.max_buy_price (station's configured threshold)
